This repository is a Python UI automation framework for Jaldee Business using:

- Python
- pytest
- Playwright
- pytest-playwright
- Allure reporting
- `.env` based configuration
- JSON test data
- function-based framework helpers

```text
F:\UI Automation\JaldeeUI.Playwright
```

The target application currently used for tests is:

```text
https://scale.jaldee.com/business/login
```

## Locator Strategy

The UI is transitioning from old Angular HTML to stable `data-testid` locators.

Use this locator priority:

```text
1. data-testid for new UI
2. Playwright role locators
3. Placeholder locators
4. Stable IDs
5. Stable form attributes such as formcontrolname
6. Text locators
7. XPath only as a last option
```

Good examples:

```python
page.get_by_test_id("sign-in-button")
page.get_by_role("button", name="Sign In")
page.get_by_placeholder("Enter Login ID")
page.locator("#loginId")
page.locator("input[formcontrolname='loginId']")
```

Avoid:

```python
page.locator("div:nth-child(3) > div > .p-element > .p-card")
page.locator("input[_ngcontent-tus-c54]")
```

Angular-generated attributes such as `_ngcontent-*` are unstable and must not be used.

## Current Login UI Selector Context

Old login ID field has stable attributes like:

```html
<input
  type="text"
  id="loginId"
  formcontrolname="loginId"
  placeholder="Enter Login ID">
```

Old password field has stable attributes like:

```html
<input
  id="password"
  placeholder="Enter password"
  formcontrolname="password"
  type="password">
```

Current selector constants should include:

```python
TEST_ID_ATTRIBUTE = "data-testid"

LOGIN_ID_TEST_ID = "login-id-input"
PASSWORD_TEST_ID = "password-input"
SIGN_IN_TEST_ID = "sign-in-button"

LOGIN_ID_SELECTOR = "#loginId"
LOGIN_ID_FORM_CONTROL_SELECTOR = "input[formcontrolname='loginId']"

PASSWORD_SELECTOR = "#password"
PASSWORD_FORM_CONTROL_SELECTOR = "input[formcontrolname='password']"
```

The login locator fallback strategy was optimized for speed:

- Check optional `data-testid` quickly, using a short timeout such as 500 ms.
- Then use old stable selectors such as `#loginId` and `#password`.
- Do not allow a missing `data-testid` to wait for the full default timeout.

## data-testid Strategy

For new UI work, developers should add stable test IDs:

```html
<input data-testid="login-id-input" />
<input data-testid="password-input" />
<button data-testid="sign-in-button">Sign In</button>
<button data-testid="booking-view-button">View</button>
<button data-testid="appointment-details-back-button">Back</button>
```

In Python Playwright, if configuring test id explicitly, use the Playwright fixture:

```python
@pytest.fixture(scope="session", autouse=True)
def configure_playwright_test_id(playwright):
    playwright.selectors.set_test_id_attribute("data-testid")
```

Do not use this invalid import:

```python
from playwright.sync_api import selectors as playwright_selectors
```

That causes:

```text
ImportError: cannot import name 'selectors' from 'playwright.sync_api'
```

## Codegen Workflow

Codegen is used only to discover flows and locators. Generated code should not be copied directly into final tests without cleanup.

Create the autogenerated folder:

```powershell
New-Item -ItemType Directory -Force -Path "F:\UI Automation\JaldeeUI.Playwright\Playwright Autogenerated"
```

Run codegen:

```powershell
playwright codegen --target python-pytest -o "F:\UI Automation\JaldeeUI.Playwright\Playwright Autogenerated\test_business_flow.py" "https://scale.jaldee.com/business/login"
```

Recommended codegen workflow:

```text
1. Use codegen to discover selectors and flow.
2. Save the generated file before closing the browser.
3. Review generated selectors.
4. Replace unstable selectors with data-testid, role, placeholder, id, or formcontrolname selectors.
5. Move reusable steps into framework action files.
6. Keep final test files short and readable.
```

Do not leave long codegen locators such as `nth-child` chains in final tests.

## What Not To Do

Do not:

- Reintroduce ReportPortal.
- Convert the framework to classes/page objects.
- Remove headed mode unless the user asks.
- Use Angular `_ngcontent-*` attributes.
- Use long CSS `nth-child` chains in final tests.
- Add broad global retries that hide real failures.
- Over-engineer fixtures before needed.
- Commit `.env`.
- Hardcode credentials in committed files.
- Generate/open stale Allure reports after pytest import failure.
