## Generate Playwright Code Using Codegen

Playwright codegen can be used to record browser actions and generate starter test code.

Create the autogenerated code folder if it does not already exist:

```powershell
New-Item -ItemType Directory -Force -Path "F:\UI Automation\JaldeeUI.Playwright\Playwright Autogenerated"

Run codegen:

playwright codegen --target python-pytest -o "F:\UI Automation\JaldeeUI.Playwright\Playwright Autogenerated\test_business_flow.py" "https://scale.jaldee.com/business/login"

This will:

Open a browser.
Start recording your actions.
Generate Python pytest Playwright code.
Save the generated code to:
F:\UI Automation\JaldeeUI.Playwright\Playwright Autogenerated\test_business_flow.py

Important notes:

Complete the flow in the opened browser.
Before closing the browser, make sure the generated file is saved.
Treat generated code as a starting point, not final framework code.
Move reusable actions from generated code into files under framework/.
Keep test files clean and call reusable functions instead of keeping long codegen locators directly in tests.
Review generated selectors carefully. Replace unstable selectors like nth-child, long CSS paths, or dynamic text with stable locators wherever possible.

Run Tests

Run all tests:

pytest

Run with browser visible:

pytest --headed

Run appointment tests:

pytest -m appointment --headed

Run auth tests:

Run smoke tests:

pytest -m smoke --headed

Run one test file:

pytest tests/test_appointment_details.py --headed

Run one test case:

pytest tests/test_appointment_details.py::test_view_first_appointment_details_from_dashboard --headed

Run tests by name:

pytest -k logout --headed
Allure Report

.\scripts\run_with_allure.ps1

Manual Allure generation:

pytest
allure generate reports/allure-results -o reports/allure-report --clean
allure open reports/allure-report
Test Data

Appointment test data is stored in:

data/appointment_test_data.json

Update known patient names there instead of hardcoding values in test files.

Adding New Tests

Use this pattern:

def test_new_flow(page, config):
    login(page, config)
    select_first_business_card(page)

    # feature actions here

    # assertions here

Reusable actions should go inside framework/.

Examples:

framework/order_actions.py
framework/invoice_actions.py
framework/customer_actions.py
framework/finance_actions.py
Reporting and Debugging

Allure results:

reports/allure-results

Generated Allure report:

reports/allure-report

Playwright screenshots, videos, and traces:

reports/artifacts

Current run JSON/JUnit/logs:

reports/current
Notes
.env should not be committed.
Use stable selectors whenever possible.
Prefer data-testid attributes when the application supports them.
Avoid time.sleep() in tests.
Use framework helper methods for clicks, fills, waits, login, and logout.
